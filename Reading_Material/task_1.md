# 👨‍🎓 Task 1: Reusable `<DataTable />` - સંપૂર્ણ અભ્યાસ માર્ગદર્શિકા (Study Material)

હેલો શિવાની! આ સ્પેશિયલ ડોક્યુમેન્ટ માત્ર તમારા વાંચવા (Reading) અને ઇન્ટરવ્યુની તૈયારી માટે બનાવવામાં આવ્યું છે. આ ફાઇલ તમારા પ્રોજેક્ટના મેઇન કોડથી અલગ `Reading_Material` ફોલ્ડરમાં સેવ કરેલી છે જેથી તમે તેને આસાનીથી શાંતિથી વાંચી શકો.

## 🎯 ૧. ટાસ્ક શું છે? (What is the Task?)

કંપનીએ તમને **React** માં એક **Reusable DataTable** બનાવવાનું કામ આપ્યું છે. 
* **Reusable** એટલે કે ટેબલનો કોડ એક જ વાર લખાયેલો હોવો જોઈએ, પણ તે ડેટાબેઝના અલગ-અલગ ડેટા (જેમ કે Students અને Products) સાથે કોઈપણ ફેરફાર વગર ઓટોમેટિક સેટ થઈને કામ કરી શકે.

### મુખ્ય જરૂરિયાતો (Core Requirements):
1. **Sorting**: કોલમના હેડર પર ક્લિક કરવાથી ડેટા ચડતા (A-Z) કે ઉતરતા (Z-A) ક્રમમાં ગોઠવાય.
2. **Search Input**: યુઝર કંઈ પણ સર્ચ કરે તો તે આખા ટેબલમાંથી ફિલ્ટર થવું જોઈએ.
3. **Pagination**: એક પેજ પર લિમિટેડ ડેટા દેખાય (બાય ડિફોલ્ટ ૧૦ રેકોર્ડ્સ) અને નીચે Next/Prev બટન્સ હોવા જોઈએ.
4. **Empty State & Loading State**: 
   * જ્યારે ડેટા લોડ થતો હોય ત્યારે સુંદર પલ્સ એનિમેશન વાળા સ્કેલેટન બાર દેખાય.
   * જો કોઈ ડેટા મેચ ન થાય, તો સુંદર આઇકોન સાથે "No matching records found" દેખાય.

## 🛠️ ૨. પ્રોજેક્ટ કઈ રીતે ક્રિએટ થયો? (How it was created?)

આપણે આ પ્રોજેક્ટ નીચે મુજબના પ્રોફેશનલ સ્ટેપ્સ ફોલો કરીને બનાવ્યો છે:

1. **Git Branch Setup**: પહેલા આપણે મેઇન બ્રાન્ચમાંથી નવો રસ્તો (બ્રાન્ચ) બનાવ્યો જેનું નામ `task-1-datatable` રાખ્યું. આનાથી આપણો ટાસ્ક ૧ નો કોડ બાકીના ટાસ્કથી સુરક્ષિત અને અલગ રહે છે.
2. **React App Generation with Vite**: આપણે નીચેના ટૂલ અને ટેમ્પલેટનો ઉપયોગ કરીને પ્રોજેક્ટ સ્ટાર્ટ કર્યો:
   ```bash
   npx create-vite@latest datatable-demo --template react --no-interactive
   ```
   * *શા માટે Vite?* કારણ કે Vite એ મોર્ડન બિલ્ડ ટૂલ છે જે જુના Create-React-App કરતાં ૧૦ ગણું ફાસ્ટ ચાલે છે અને ડેવલપમેન્ટમાં ખૂબ જ સરળ છે.
3. **Dependency Installation**: પ્રોજેક્ટ શરૂ કરવા માટે આપણે `npm install` કમાન્ડ રન કર્યો જેથી રીએક્ટની બેકગ્રાઉન્ડ ફાઇલો ઇન્સ્ટોલ થાય.
4. **Component Design**: આપણે `src/components` ફોલ્ડર બનાવીને તેની અંદર `DataTable.jsx` અને `DataTable.css` ફાઇલ લખીને આખું લોજીક રેડી કર્યું.

## 📂 ૩. કઈ ફાઇલ અને ફોલ્ડરનું શું કામ છે? (File & Folder Breakdown)

જ્યારે તમે `Task_1/datatable-demo` ફોલ્ડર ઓપન કરો છો, ત્યારે નીચેની ફાઇલો જોવા મળે છે. દરેકનું કામ અહી સરળતાથી સમજાવ્યું છે:

### 📁 Root Project Level (કન્ફિગ્યુરેશન ફાઇલો)
* 📄 **`package.json`**:
  * **કાર્ય**: આ પ્રોજેક્ટની મેઈન ફાઇલ છે. આમાં પ્રોજેક્ટમાં વપરાતી બધી જ લાઇબ્રેરીઓની યાદી (Dependencies) અને કમાન્ડ્સ (જેમ કે `npm run dev`) લખેલા હોય છે.
* 📄 **`vite.config.js`**:
  * **કાર્ય**: આ વાઇટ (Vite) ની સેટિંગ ફાઇલ છે. આ સર્વર કઈ રીતે ચાલશે અને બિલ્ડ કઈ રીતે બનશે તે નક્કી કરે છે.
* 📄 **`index.html`**:
  * **કાર્ય**: આ આખી એપ્લિકેશનનું મેઈન વેબ પેજ છે. આમાં એક `<div id="root"></div>` હોય છે, જેની અંદર આપણો આખો રીએક્ટનો કોડ ઓટોમેટિક લોડ થાય છે.

### 📁 `src/` Folder (જ્યાં બધો જ સોર્સ કોડ છે)
* 📄 **`main.jsx`**:
  * **કાર્ય**: આ રીએક્ટનો એન્ટ્રી ગેટ છે. આ ફાઇલ સૌથી પહેલા રન થાય છે અને આપણી `App.jsx` ફાઇલને `index.html` ના `root` ડિવાઇઝમાં માઉન્ટ (લોડ) કરે છે.
* 📄 **`index.css`**:
  * **કાર્ય**: ગ્લોબલ સીએસએસ સ્ટાઇલ. આમાં આપણે બ્રાઉઝરનું વધારાનું માર્જિન દુર કર્યું છે અને સ્ક્રોલબારને સુંદર બનાવ્યો છે.
* 📄 **`App.jsx`**:
  * **કાર્ય**: આ આપણા ડેશબોર્ડનું મેઇન કમ્પોનન્ટ છે. આમાં આપણે:
    1. Students અને Products ના ડમી ડેટા સેવ કર્યા છે.
    2. ટેબલમાં કઈ કઈ કોલમ બતાવવી તેનું લિસ્ટ (Column Config) બનાવ્યું છે.
    3. ઉપર દેખાતા "Students Demo" અને "Products Demo" ના ટૅબ બટન્સ અને લોડિંગ સ્ટેટ ચેક કરવાના બટન્સનું કોડિંગ કર્યું છે.
* 📄 **`App.css`**:
  * **કાર્ય**: આ `App.jsx` પેજને લક્ઝરી અને મોર્ડન બ્લેક-પર્પલ લુક આપવા માટેની ડિઝાઇન ફાઇલ છે.

### 📁 `src/components/` Folder (તમારો મુખ્ય મેજીક કોડ)
* 📄 **`DataTable.jsx`**:
  * **કાર્ય**: આમાં જ આખા ટેબલનું એન્જિન (લોજીક) ફિટ કરેલું છે. આ ફાઇલમાં:
    * **`filteredData`**: ગ્લોબલ સર્ચ બોક્સમાં લખાયેલ અક્ષરોને આખા ડેટામાંથી ફિલ્ટર કરતું ફંક્શન.
    * **`sortedData`**: કોલમના હેડર પર ક્લિક કરવાથી લિસ્ટને સોર્ટ કરતું ફંક્શન.
    * **`paginatedData`**: ૧૦,૦૦૦ માંથી માત્ર ૧૦ જ રેકોર્ડ સ્ક્રીન પર બતાવીને બાકીના પેજ વાઈઝ સેટ કરતું ગણિત.
* 📄 **`DataTable.css`**:
  * **કાર્ય**: આ ફાઇલ ટેબલને કાચ જેવો ટ્રાન્સપરન્ટ લુક (Glassmorphism), એક્ટિવ પેજના કલર, અને જ્યારે લોડ થતો હોય ત્યારે પલ્સિંગ બાર (Shimmer effect) વાળા એનિમેશન આપે છે.

## ⚡ ૪. લોજીક કઈ રીતે કામ કરે છે? (Deep Logic Explanation)

તમારે કોડની અંદરનું લોજીક સમજવું જરૂરી છે:

### 1. Reusability (રીયુઝેબિલિટી):
આપણે `App.jsx` માં ટેબલને આ રીતે કોલ કરીએ છીએ:
```javascript
<DataTable columns={studentColumns} data={MOCK_STUDENTS} />
```
અહીં `columns` માં આપણે કહીએ છીએ કે ટેબલમાં કઈ કઈ કોલમ બનશે, અને `data` માં ડેટા મોકલીએ છીએ. ટેબલ તેના `columns.map()` લૂપ ચલાવીને હેડર અને રો ઓટોમેટિક ક્રિએટ કરી દે છે.

### 2. Global Search:
રીએક્ટમાં સર્ચ કઈ રીતે ફાસ્ટ ચાલે છે? આપણે `useMemo` વાપર્યું છે. તે આખા ઓબ્જેક્ટના બધા જ ખાનાઓની કિંમતો (Values) ને ટેક્સ્ટમાં કન્વર્ટ કરીને ચેક કરે છે કે તેમાં સર્ચ કરેલો શબ્દ છે કે નહીં:
```javascript
const filteredData = data.filter(row => 
  columns.some(col => String(row[col.key]).toLowerCase().includes(query))
);
```

### 3. sorting (3-Way Sorting):
આપણે સોર્ટિંગને ૩ સ્ટેપમાં સેટ કર્યું છે:
1. **પહેલી ક્લિક**: ડેટા એસેન્ડિંગ (A ➡️ Z) થશે.
2. **બીજી ક્લિક**: ડેટા ડીસેન્ડિંગ (Z ➡️ A) થશે.
3. **ત્રીજી ક્લિક**: સોર્ટિંગ બંધ થઈ જશે (Neutral) અને ડેટા ઓરીજીનલ ક્રમમાં પાછો આવી જશે.

## 🔥 ૬. ૨-મિનિટ ક્વિક રિવિઝન ફોર ઇન્ટરવ્યુ (2-Minute Cheat-Sheet)

જ્યારે ઇન્ટરવ્યુઅર પૂછે કે **"તમે આ ટાસ્ક ૧ માં શું કર્યું છે?"**, ત્યારે તમારે મોઢે રાખવાની શોર્ટ સમરી:

1. **ટાસ્કનો ગોલ (The Goal)**: 
   * *"મેં React (Vite) માં એક સિંગલ, પર્ફોર્મન્ટ અને **Reusable DataTable** બનાવ્યું છે, જે કોઈપણ ડેટા શેપ સાથે ચાલી શકે."*
2. **મુખ્ય ફીચર્સ (Key Features)**:
   * **Global Search**: પૃષ્ઠભૂમિમાં `useMemo` વાપરીને આખા ડેટા ઓબ્જેક્ટ્સ પર રિયલ-ટાઇમ અક્ષર-બાય-અક્ષર ફિલ્ટરિંગ.
   * **3-Way Sorting**: કોલમ હેડર ક્લિકથી ASC ➡️ DESC ➡️ None (Neutral) સોર્ટિંગ.
   * **Pagination**: ક્લાયન્ટ-સાઇડ પેજીનેશન, કન્ફિગ્યુરેબલ રો-સાઇઝ (૫, ૧૦, ૨૦, ૫૦) સાથે.
3. **ડિઝાઇન થીમ (Aesthetics)**:
   * *"મેં આખી એપ્લિકેશન **Premium Dark Theme** અને **Glassmorphic** લુક સાથે ડિઝાઇન કરી છે. પર્ફોર્મન્સ ટેસ્ટ કરવા માટે મેં 'Simulate Loading Skeletons' અને 'Test Empty State' ના લાઈવ બટન્સ આપ્યા છે."*
4. **કોડની વિશેષતા (Technical Highlights)**:
   * **useMemo Optimization**: સોર્ટિંગ અને ફિલ્ટરિંગના ફંક્શન્સ `useMemo` થી લૉક કર્યા છે જેથી ફાલતુ રી-રેન્ડરીંગ અટકે.
   * **Custom Renderers**: કોલમ કન્ફિગમાં મેં કસ્ટમ ફંક્શન સપોર્ટ આપ્યો છે જેથી રેટિંગ્સ, પ્રોગ્રેસ બાર્સ કે કલરફુલ સ્ટેટસ બેજીસ ટેબલમાં ડાયરેક્ટ દેખાઈ શકે.
   * **Automated QA Ready**: બધા બટનો અને ઇનપુટ્સમાં યુનિક **IDs** એડ કર્યા છે જેથી ટેસ્ટિંગ ઓટોમેશન સરળ બને.
